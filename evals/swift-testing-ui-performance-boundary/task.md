# Swift Testing Boundary Plan

## Problem/Feature Description

A team is planning tests for a SwiftUI checkout flow. They need to unit-test a price calculator, verify async coupon events, snapshot the checkout screen in dark mode and large Dynamic Type, measure parser performance, and automate the login-to-checkout UI path.

## Output Specification

Create `swift-testing-boundary-plan.md` that assigns each test category to Swift Testing, XCTest/XCUITest, or snapshot tooling as appropriate, and includes a migration-oriented file organization plan. Do not create an Xcode project.
