# Pre-Submission Ownership Split

## Problem/Feature Description

An iOS team is trying to finish the App Store submission checklist, but one planning doc mixes product-page growth work with privacy, tracking, subscription, and rejection-response concerns. They need a clean ownership split across `app-store-optimization`, `app-store-review`, and `storekit` so the right specialist can take each part.

The doc includes keyword ideas, App Store description copy, screenshot captions, custom pages for campaigns, `PrivacyInfo.xcprivacy` entries, ATT purpose wording, subscription paywall code, StoreKit entitlement checks, and a possible metadata rejection because screenshots show marketing mockups rather than real UI.

## Output Specification

Create `ownership.md` that assigns each concern to the right skill/domain and explains what `app-store-optimization` should still do before submission.
